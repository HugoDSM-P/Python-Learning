# Day 12 – GUI Development with Tkinter (Restaurant Billing System)

Today I built a complete desktop application using `Tkinter`, focusing on GUI design, event handling, and application logic.

## What I learned

### Tkinter Basics

- Creating windows and frames
- Organizing layouts using `Frame`, `LabelFrame`, `Label`, `Entry`, and `Button`
- Building structured GUI applications

### Event Handling

- Using functions to handle button clicks
- Linking UI elements with logic using `command`
- Managing user interactions in real time

### Dynamic UI Behavior

- Enabling and disabling input fields
- Using `Checkbutton` with `IntVar`
- Updating values dynamically based on user selection

### Data Processing

- Calculating totals, subtotals, and taxes
- Using lists to manage multiple items
- Formatting and displaying results in the UI

### File Handling

- Saving receipts to external `.txt` files
- Using `filedialog` to choose save location
- Displaying confirmation messages with `messagebox`

### Extra Features

- Built-in calculator using GUI buttons
- Receipt generator with timestamps and random order numbers
- Reset system to clear all data and restart the app

## Project – Restaurant Billing System

I developed a full desktop application that:

- Lets users select food, drinks, and desserts
- Calculates total price including taxes
- Generates a detailed receipt
- Includes a working calculator inside the app
- Allows saving receipts as text files
- Resets all inputs for a new order

## Summary

This project helped me combine multiple Python concepts into a single real-world application, including GUI development, event-driven programming, and data management.
